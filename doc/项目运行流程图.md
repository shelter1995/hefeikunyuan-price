# 项目运行流程图

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                     报价更新系统                              │
│                  (ocr_price + skills)                        │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
       ┌──────────┐   ┌──────────┐   ┌──────────┐
       │  网价链路  │   │ 图片/文档 │   │  库存标注 │
       │  (web)   │   │  (image) │   │ (color)  │
       └────┬─────┘   └────┬─────┘   └────┬─────┘

            ▼              ▼              ▼
       更新G1/G3/G4   更新H1/H3/H4   标注单元格颜色
```

## 详细流程

### 1. 网价链路（web_price.py）

```
用户输入
    │
    ▼
提取地点信息（从文件名）
    │
    ▼
登录网价站点（Playwright）
    │
    ▼
查询最新价格清单
    │
    ▼
生成清单Excel
    │
    ▼
加载/复用厂家对照表
    │
    ├─ 存在新厂家？→ 【阻断】展示明细 → 用户确认 → 写入已确认
    │
    ▼
回写项目文件
    │
    ├─ G1: 日期
    ├─ G3: 盘螺价格
    └─ G4: 螺纹价格
    │
    ▼
页签处理：先清空 → 标红未匹配
```

### 2. 图片/文档链路（pipeline.py + minimax_vision.py）

```
用户输入
    │
    ▼
收集线下报价文件
    │
    ├─ .jpg/.png/.pdf → MiniMax视觉识别（提取价格+库存）
    ├─ .txt           → 文本解析（提取价格+库存）
    └─ 补充.txt       → 解析调整规则（下30/电议等）
    │
    ▼
生成JSON结果文件
    │
    ▼
加载/复用图片文档对照表
    │
    ├─ 存在新厂家？→ 【阻断】展示明细 → 用户确认 → 写入已确认
    │
    ▼
回写项目文件
    │
    ├─ H1: 日期
    ├─ H3: 盘螺价格
    └─ H4: 螺纹价格
    │
    ▼
【禁止】修改页签颜色
```

### 3. 库存颜色标注链路（inventory_color.py）

```
价格回写完成后
    │
    ▼
清空"报价表"sheet原有颜色
    │
    ▼
建立映射
    │
    ├─ 厂家名 → 列号（读取第1行）
    │   ├─ 徐钢 → P列
    │   ├─ 闽源 → R列
    │   ├─ 长江 → T列(厂内) / V列(蚌埠)
    │   ├─ 金虹 → AD列
    │   ├─ 贵航 → AJ列
    │   └─ 桂鑫 → AN列
    │
    └─ 规格+长度 → 行号（读取A-C列）
        ├─ (一级钢, 6, null) → 9行（盘螺6）
        ├─ (一级钢, 8, null) → 10行（盘螺8）
        ├─ (抗震三级钢, 10, 9) → 15行（螺纹10-9米）
        ├─ (抗震三级钢, 12, 9) → 16行（螺纹12-9米）
        └─ ...
    │
    ▼
逐厂家标注颜色
    │
    ├─ 充足 → 🔵蓝色（FF0070C0）
    ├─ 告警 → 🟡黄色（FFFFC000）
    └─ 缺货 → 🔴红色（FFFF0000）
    │
    ▼
只标第一列（价格列），不标网差列
```

## 文件处理流程

### 图片文件处理

```
线下报价/XX报价.jpg
    │
    ▼
MiniMax视觉识别（analyze_quote_image_to_ocr_format）
    │
    ├─ 3次重试+合并机制
    │   ├─ 价格：取非null值
    │   └─ 状态：缺货 > 告警 > 充足
    │
    ▼
生成JSON：运行产物/ocr价格提取_XX报价.json
    │
    ├─ company: 厂家名
    ├─ quote_date: 报价日期
    ├─ records: [价格记录]
    │   ├─ location: 城市
    │   ├─ rebar_price: 螺纹价格
    │   └─ coil_price: 盘螺价格
    └─ _vision_result: {库存情况}
        └─ 库存情况: [{规格, 状态, 原始描述}]
```

### 文本文件处理

```
线下报价/XX报价.txt
    │
    ▼
文本解析（parse_price_lines_from_text）
    │
    ├─ 提取价格
    │   ├─ 螺纹价格
    │   └─ 盘螺价格
    │
    └─ 提取库存（parse_inventory_from_text）
        ├─ "9米HRB400E规格有：12、14（少）、16"
        ├─ "盘螺规格有：6、8、10（少）、12"
        └─ "螺纹 抗震9米：10无货、12、14"
    │
    ▼
生成JSON：运行产物/ocr价格提取_XX报价.json
```

### 补充文件处理

```
线下报价/XX报价补充.txt
    │
    ▼
解析调整规则
    │
    ├─ "蚌埠下30" → 基础价 - 30
    ├─ "合肥电议" → 价格=null，标记电议
    └─ "蚌埠：3195螺纹" → 直接采用，覆盖图片价
    │
    ▼
优先级：补充文件 > 图片价格
```

## 执行命令流程

### 单文件更新

```
用户：更新项目报价/XX询价表.xlsx
    │
    ▼
解析：mode=both（默认）
    │
    ▼
执行：python skills/quote-update/scripts/run_single.py --project "项目报价/XX询价表.xlsx" --mode both --headless
    │
    ├─ Step 1: 网价链路
    │   ├─ 登录站点
    │   ├─ 查询价格
    │   ├─ 生成清单
    │   ├─ 对照确认（如有新厂家）
    │   └─ 回写G列
    │
    ├─ Step 2: 图片/文档链路
    │   ├─ 收集文件
    │   ├─ MiniMax识别/文本解析
    │   ├─ 对照确认（如有新厂家）
    │   ├─ 回写H列
    │   └─ 库存颜色标注
    │
    └─ Step 3: 生成报告
        ├─ 网价更新明细
        ├─ 图片/文档更新明细
        ├─ 库存颜色统计
        └─ 汇总统计
```

### 批量更新

```
用户：批量更新项目报价文件夹
    │
    ▼
执行：python skills/quote-update/scripts/run_batch.py --project-dir "项目报价" --glob "*.xlsx" --mode both --headless
    │
    ├─ 扫描文件夹
    ├─ 逐个处理每个xlsx文件
    │   ├─ 网价链路
    │   └─ 图片/文档链路
    │
    └─ 生成批量报告
        ├─ 每个项目的更新明细
        └─ 汇总统计表
```

## 错误处理流程

### MiniMax API失败

```
MiniMax API调用
    │
    ├─ 第1次尝试
    │   ├─ 成功 → 继续
    │   └─ 失败 → 第2次尝试
    │
    ├─ 第2次尝试
    │   ├─ 成功 → 继续
    │   └─ 失败 → 第3次尝试
    │
    ├─ 第3次尝试
    │   ├─ 成功 → 继续
    │   └─ 失败 → 记录错误
    │
    └─ 3次都失败
        ├─ 记录错误日志
        ├─ 跳过该图片
        └─ 报告中提示"识别失败，使用上回价格"
```

### 待确认阻断

```
执行过程中
    │
    ├─ 发现新厂家
    │   ├─ 网价链路：新厂家未在对照表中
    │   └─ 图片/文档链路：新厂家未在对照表中
    │
    ▼
【阻断】停止执行
    │
    ▼
展示待确认明细
    │
    ├─ 网价对照明细（如适用）
    │   ├─ 项目Sheet
    │   ├─ 网价厂家Sheet
    │   ├─ 匹配分数
    │   └─ 状态
    │
    └─ 图片/文档对照明细（如适用）
        ├─ 项目Sheet
        ├─ 来源Sheet
        ├─ 匹配分数
        └─ 状态
    │
    ▼
等待用户确认
    │
    ├─ "确认全部" → 全部写入已确认
    ├─ 逐条确认 → 按用户指示写入
    └─ 取消 → 终止执行
    │
    ▼
写入已确认文件
    │
    ▼
运行check_pending_mapping.py确认待确认为0
    │
    ▼
重新执行原命令
```
