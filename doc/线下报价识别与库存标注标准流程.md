# 线下报价识别与库存标注标准流程

## 版本：1.2
## 日期：2026-04-30
## 适用：所有项目报价文件的网价+线下价+库存更新

> 2026-05-11 补充：图片主链路为 `MiniMax VLM结构化识别 → Python业务规则校验 → 厂家对照人工确认 → Excel回写`。传统 OCR 仅保留为历史诊断脚本，不参与默认执行链路。

---

## 一、总体流程

```
项目文件(.xlsx)
    │
    ├── Step 1: 查找网价 ──────────→ 更新项目文件"网价Sheet"
    │       (web_price.py)
    │
    ├── Step 2: 查找线下报价 ──────→ 生成JSON价格文件
    │   ├── 图片文件(.jpg/.png) ──→ MiniMax视觉识别（提取价格+库存）
    │   ├── 文本文件(.txt) ──────→ 文本解析（提取价格+库存）
    │   └── 补充文件(补充.txt) ───→ 优先级规则合并
    │       (writeback_image_doc.py + minimax_vision.py + parser.py)
    │
    ├── Step 3: 【新增】厂家名称对照表确认
    │       ├── 读取报价表第1行 → 提取厂家名+列号
    │       ├── 读取JSON文件 → 提取厂家名
    │       ├── 生成对照表 → 打印到控制台
    │       └── 等待用户确认映射正确
    │       (pipeline.py: _build_factory_mapping + _print_factory_mapping)
    │
    ├── Step 4: 更新价格 ──────────→ 更新"报价表"Sheet价格单元格
    │       (writeback_image_doc.py)
    │
    ├── Step 5: 更新库存颜色 ──────→ 标注"报价表"Sheet单元格颜色
    │       (inventory_color.py)
    │
    └── Step 6: 输出结果概述 ──────→ 控制台报告
            (pipeline.py)
```

价格偏差校验：线下报价写入 H3/H4 前，必须读取同一厂家 sheet 的 G3/G4 网价作为参考。盘螺使用 H3 vs G3，螺纹使用 H4 vs G4。任一维度超过 1000 元/吨或 20% 偏离时，本厂家本次线下价不自动写入，进入报告待人工确认。没有 G3/G4 参考价时不阻断，但报告必须标注“无网价参考”。

## 二、Step 2 详细流程：查找线下报价

### 2.1 文件分类

线下报价文件夹(`线下报价/`)中的文件分为三类：

| 文件类型 | 扩展名 | 处理方式 | 输出 |
|---------|--------|---------|------|
| 报价图片 | .jpg / .png | MiniMax VLM视觉识别 | JSON（含价格+库存） |
| 报价文本 | .txt（无"补充"字样） | 文本解析（价格+库存） | JSON（含价格+库存） |
| 补充文本 | .txt（含"补充"字样） | 文本解析 | 调整规则（下30/电议等） |

### 2.2 图片处理流程（MiniMax视觉识别）

#### 2.2.1 调用API

```python
from ocr_price.minimax_vision import analyze_quote_image_to_ocr_format

result = analyze_quote_image_to_ocr_format(
    image_path="线下报价/XX报价.png",
    target_cities=["合肥", "蚌埠"],  # 根据文件名解析
    use_retry=True,  # 3次重试+合并
)
```

#### 2.2.2 输出JSON结构

```json
{
  "company": "厂家名称",
  "quote_date": "2026-04-24",
  "records": [
    {
      "location": "蚌埠",
      "rebar_price": 3210,
      "coil_price": 3390,
      "rebar_raw": "3210",
      "coil_raw": "3390"
    }
  ],
  "_vision_result": {
    "库存情况": [
      {"规格": "9米螺纹14E", "状态": "告警", "原始描述": "极少"}
    ]
  }
}
```

#### 2.2.3 库存状态映射

| 视觉识别状态 | 报价表颜色 | 触发条件 |
|------------|-----------|---------|
| 充足 | 🔵蓝色 | 规格后无说明文字 |
| 告警 | 🟡黄色 | "极少"、"少"、"XX件"、"XX配" |
| 缺货 | 🔴红色 | "无货"、"无"、"等生产" |

**重要规则**：
- "XX件"（如82件、166件）**必须**标为告警，不是充足
- "XX配"（如22E配、25E配）**必须**标为告警
- 规格后无任何文字 = 充足（蓝色）

#### 2.2.4 图片价格提取原则

**核心原则**：以报价区域（表格）内显示的价格为准，不自行根据差值描述计算。

| 场景 | 处理方式 | 示例 |
|------|---------|------|
| 报价表格中有明确价格 | 直接取表格值 | 合肥=3230 → 取3230 |
| 图片上方/下方有差值描述 | 忽略差值，取表格值 | "合肥+220"忽略，取表格值 |
| 报价区域某城市价格=0或空，上方有差值描述 | 价格置空，报告中说明 | 合肥=0 → null，报告注明 |
| 补充.txt中有调整规则 | 必须应用 | "蚌埠下30" → 基础价-30 |

**报价区域价格为0或空时的通用处理**：
- 当任意厂家的报价区域某城市价格显示为0或空，而图片上方/下方有差值描述（如"合肥+220"）时：
  - 输出：该城市价格=null
  - 报告中说明："XX厂家XX区域报价为0/空，上方差值描述为XXX+XX，价格置空，需人工确认"
  - 禁止自行用基础价+差值计算

**差值描述 vs 补充文件**：
- 图片中的差值描述 → **不应用**，仅取表格值
- 补充.txt文件的调整规则 → **必须应用**，优先级高于图片价格

### 2.3 文本文档处理流程

#### 2.3.1 普通文本文件（非补充文件）

```python
from ocr_price.parser import parse_price_lines_from_text, parse_inventory_from_text

raw_text = Path("线下报价/闽源报价.txt").read_text(encoding="utf-8")

# 提取价格
price_result = parse_price_lines_from_text(raw_text, target_location="蚌埠")

# 提取库存
inventory_result = parse_inventory_from_text(raw_text)
```

**文本文件库存提取规则**：

| 文本格式示例 | 提取结果 |
|-------------|---------|
| `9米HRB400E规格有：12、14（少）、16` | 9米螺纹12=充足, 9米螺纹14=告警, 9米螺纹16=充足 |
| `盘螺规格有：6、8、10（少）、12` | 盘螺6=充足, 盘螺8=充足, 盘螺10=告警, 盘螺12=充足 |
| `螺纹 抗震9米：10无货、12、14` | 9米螺纹10=缺货, 9米螺纹12=充足, 9米螺纹14=充足 |
| `12规格明天生产` | 螺纹12=缺货 |

#### 2.3.2 补充文件处理

补充文件优先级**高于**图片价格：

| 补充内容 | 处理方式 | 示例 |
|---------|---------|------|
| "蚌埠下30" | 图片基础价 - 30 | 3210 - 30 = 3180 |
| "合肥电议" | 价格设为null，标记电议 | 不更新价格，提示"电议" |
| "蚌埠：3195螺纹、3455盘螺" | 直接采用，覆盖图片价 | 螺纹3195，盘螺3455 |

### 2.4 金虹特殊规则

- 金虹厂家**默认提取螺纹14**（其他厂家提取螺纹18）
- 如果图片中无螺纹14价格 → 设为null（不强制提取）

### 3.3 特殊厂家：长江双列映射

长江厂家在报价表中**占用4列**：
- **T列** = 厂内库存（图片中以"厂内"开头的库存记录）
- **U列** = 厂内网差
- **V列** = 蚌埠厂库库存（图片中以"蚌埠"开头的库存记录）
- **W列** = 蚌埠厂库网差

库存标注时根据前缀选择列：
- "厂内9m螺纹12E" → 标注 **T列**
- "蚌埠9m螺纹12E" → 标注 **V列**
- "蚌埠盘螺8E" → 标注 **V列**

用户确认映射正确后，pipeline继续执行：

    ("抗震三级钢", "12", "9"): 16,  # 螺纹12-9米

#### 4.2.3 规格归一化匹配

库存识别出的规格 → 报价表列匹配逻辑：

```python
def match_inventory_to_row(inventory_spec):
    """
    库存规格示例：
    - "9米螺纹14E" → 匹配 B=14, C=9 → Row 18
    - "盘螺8E" → 匹配 A=一级钢, B=8 → Row 10
    - "厂内9m螺纹12E" → 匹配 B=12, C=9 → Row 16（前缀"厂内"只影响列选择）
    """
    # 1. 提取数字（规格号）
    numbers = re.findall(r'\d+', inventory_spec)
    spec_number = numbers[0]
    
    # 2. 提取长度（9或12）
    length = None
    if '9米' in inventory_spec or '9m' in inventory_spec:
        length = '9'
    elif '12米' in inventory_spec or '12m' in inventory_spec:
        length = '12'
    
    # 3. 判断类型
    if '盘螺' in inventory_spec:
        steel_type = '一级钢'
    else:
        steel_type = '抗震三级钢'
    
    # 4. 查找匹配行
    return inventory_rows.get((steel_type, spec_number, length))
```

#### 4.2.4 颜色标注

```python
def apply_inventory_color(sheet, factory_col, inventory_items):
    COLORS = {
        "充足": "FF0070C0",  # 蓝色
        "告警": "FFFFC000",  # 黄色
        "缺货": "FFFF0000",  # 红色
    }
    
    for item in inventory_items:
        row = match_inventory_to_row(item.spec)
        if row is None:
            continue  # 报价表无此规格，忽略
        
        # 只标注第一列（价格列），不标网差列
        cell = sheet[f"{factory_col}{row}"]
        cell.fill = PatternFill(
            start_color=COLORS[item.status],
            end_color=COLORS[item.status],
            fill_type="solid"
        )
```
