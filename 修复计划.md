### 1.1 修复Unicode编码冲突

**文件：** `ocr_price/pipeline.py`

**问题：** 第505行使用 `✓`（`\u2713`）和 `⚠`（`\u26a0`），Windows终端GBK编码无法处理，导致 `UnicodeEncodeError`

**修复内容：**
- 将 `✓` 替换为 `[OK]`
- 将 `⚠` 替换为 `[WARN]`
- 扫描整个代码库，确保无其他Unicode特殊字符

**代码位置：**
```python
# 原代码（第505行附近）
print(f"{item['报价表厂家']:<12} {item['JSON厂家']:<20} {item['映射列']:<30} {item['状态']}")

# 修复后
status = item['状态'].replace('\u2713', '[OK]').replace('\u26a0', '[WARN]')
print(f"{item['报价表厂家']:<12} {item['JSON厂家']:<20} {item['映射列']:<30} {status}")
```

### 1.4 修复库存颜色标注（厂家名匹配逻辑）

**文件：** `ocr_price/inventory_color.py`

**问题：** `_normalize_factory_name` 方法只提取前2个字，"安徽航鑫-贵航"→"安徽"，无法匹配"贵航"

**修复内容：**
- 修改 `_normalize_factory_name` 方法：
  - 优先使用 `FACTORY_NAME_MAP` 映射
  - 对于"XX-YY"格式，提取后半部分（如"贵航"）
  - 使用厂家对照表文件中的映射关系

**代码位置：**
```python
# 原代码（第22-33行）
def _normalize_factory_name(name: str) -> str:
    name = name.strip()
    if not name:
        return ""
    # 使用映射表
    for key, value in FACTORY_NAME_MAP.items():
        if key in name:
            return value
    # 默认取前两个字
    return name[:2]

# 修复后
def _normalize_factory_name(name: str, mapping: dict[str, str] | None = None) -> str:
    name = name.strip()
    if not name:
        return ""
    # 优先使用传入的厂家对照表映射
    if mapping:
        for key, value in mapping.items():
            if key in name or value in name:
                return value
    # 使用内置映射表
    for key, value in FACTORY_NAME_MAP.items():
        if key in name:
            return value
    # 对于"XX-YY"格式，提取后半部分
    if "-" in name:
        parts = name.split("-")
        if len(parts) >= 2:
            return parts[-1].strip()[:2]
    # 默认取前两个字
    return name[:2]
```
