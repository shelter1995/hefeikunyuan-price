# 常用命令

## 目录约定（默认）
- 项目报价表：`项目报价/*.xlsx`
- 线下报价原始文件：`线下报价/*`（jpg/png/pdf/txt/json）
- 运行产物：`运行产物/<项目报价表名>/...`

## 前置检查（必须执行）

执行任何命令前，必须先确认实际文件名：
```powershell
# 确认项目报价目录下的实际文件名
Get-ChildItem "项目报价/" | Select-Object Name
```
**严禁**将指令中的 `【】` 标记带入实际文件路径。

## 命令执行纪律

- 命令失败时（非零退出码）：必须报告完整错误、分析原因、提出修复方案、**等待用户确认**。
- 网价登录失败、验证码/滑块、登录入口不可见时：停止重跑 headless，改用不带 `--headless` 的有头 dry-run，让用户手动登录。
- 【禁止】绕过主流程脚本直接调用子命令。
- 【禁止】命令失败后编造数据或伪造报告。
- 修复后必须**重新运行同一个命令**，而非手动拼凑。

## 单文件
```powershell
python skills/quote-update/scripts/run_single.py `
  --project "项目报价/<项目Excel路径>" `
  --mode both `
  --headless
```
说明：未显式传 `--image-inputs/--image-jsons` 时，会自动读取 `线下报价` 目录。

## 单文件（网页登录需要人工辅助时）
```powershell
python skills/quote-update/scripts/run_single.py `
  --project "项目报价/<项目Excel路径>" `
  --mode both `
  --dry-run `
  --manual-login-timeout 300
```

## 批量
```powershell
python skills/quote-update/scripts/run_batch.py `
  --project-dir "项目报价" `
  --glob "*.xlsx" `
  --mode both `
  --headless
```
说明：未传 `--image-source-map` 时，会默认把 `线下报价` 全部文件作为每个项目的图片/文档来源。

## 批量（自定义映射，可选）
```powershell
python skills/quote-update/scripts/run_batch.py `
  --project-dir "项目报价" `
  --glob "*.xlsx" `
  --mode both `
  --image-source-map "skills/quote-update/assets/image_source_map.sample.json" `
  --headless
```

## 检查待确认
```powershell
python skills/quote-update/scripts/check_pending_mapping.py `
  --mapping-json "运行产物/<项目报价表名>/厂家对照表_安徽合肥_待确认.json"
```
