# 工作流程

## 执行链路概览

| 链路 | 执行内容 | 关键步骤 |
|------|---------|---------|
| `web` | 网价链路 | 登录 → 抓取 → 对照 → 回写 → 页签颜色 |
| `image_doc` | 图片/文档链路 | 收集 → 提取 → 对照 → 回写 → **库存颜色** |
| `both` | 全部链路 | 先web后image_doc |

---

## 完整流程（mode=both）

### 安全执行顺序

任何模型执行更新时必须先跑 dry-run：

```powershell
python skills/quote-update/scripts/run_single.py --project "<项目文件>" --mode both --dry-run --headless
```

如果网价登录失败、验证码/滑块阻断或登录入口改版，停止重试 headless，改用有头 dry-run：

```powershell
python skills/quote-update/scripts/run_single.py --project "<项目文件>" --mode both --dry-run
```

有头模式会在自动登录失败后等待用户手动登录，默认等待 180 秒；需要更长时间时追加 `--manual-login-timeout 300`。用户完成登录后，脚本会继续执行并保存登录态。

确认无待确认、无偏差异常，并经用户确认后，才允许真实写入：

```powershell
python skills/quote-update/scripts/run_single.py --project "<项目文件>" --mode both --confirm-write --headless --manifest "运行产物/<项目名>/dry_run_manifest.json"
```

如果 dry-run 没有输出 `Manifest:`，不得执行 confirm-write。

`--confirm-write` 默认复用 dry-run 产物直接 apply。仅在需要重抓网价或重跑OCR时，才追加：
- `--refresh-web-artifacts`
- `--refresh-image-artifacts`

### 步骤1：前置准备
1. **文件路径确认**：用 `Glob` 或 `LS` 工具确认 `项目报价/` 目录下的**实际文件名**，以磁盘上的名字为准，禁止增删任何字符。
2. 从文件名提取 `地点1`（用于网价）和 `地点2`（用于图片/文档）。
3. 创建运行产物目录：`运行产物/<项目报价表名>/`。

### 步骤2：网价链路（mode=web 或 both 时执行）

**【命令执行规则】任何命令以非零退出码结束时，必须报告完整错误、分析原因、提出方案、等待用户确认。禁止绕过主流程脚本，禁止编造数据。**

```
2.1 登录网价站点
2.2 查询最新价格清单（按日期+时间排序）
2.3 生成清单Excel：[地点][日期]建筑钢材原料价格清单.xlsx
2.4 加载/复用厂家对照表
2.5 【阻断点】存在新厂家待确认？
    - 是 → 展示明细 → 用户确认 → 写入已确认 → check_pending_mapping.py确认
    - 否 → 继续
2.6 回写项目文件：G1(日期) / G3(盘螺) / G4(螺纹)
    - 【部分更新】如果只提取到盘螺或螺纹其中一个价格，另一个为空：
      → 有值的价格必须写入，缺失的保留原值不动
      → 禁止将None/空写入覆盖旧价格
      → 只有盘螺和螺纹都为空时才跳过整个厂家
2.7 页签处理：先清空后标红未匹配
```

### 步骤3：图片/文档链路（mode=image_doc 或 both 时执行）

**【命令执行规则】同上。如果 run_single.py 失败，修复后重新运行同一个命令，禁止手动拼凑子命令。**

```
3.1 收集线下报价文件（线下报价/*.jpg *.png *.txt）
3.2 【本次修改】提取价格和库存
    ├─ .jpg/.png → MiniMax视觉识别（替代原OCR）
    │             输出：价格 + 库存状态（充足/告警/缺货）
    │   【规格规则】金虹→螺纹14，其他厂家→螺纹18
    │   【价格提取原则】以报价表格价格为准，忽略图片上/下方差值描述
    │   【价格为0或空时】置空不计算，在报告中说明差值描述供人工确认
    ├─ .txt      → 正则解析价格
    └─ 补充.txt  → 解析调整规则（下30/电议等）
3.3 生成JSON结果文件（运行产物/ocr价格提取_*.json）
3.4 加载/复用图片文档对照表（已有自动复用）
3.5 【阻断点】存在新厂家待确认？
    - 是 → 展示明细 → 用户确认 → 写入已确认 → check_pending_mapping.py确认
    - 否 → 继续
3.6 回写项目文件：H1(日期) / H3(盘螺) / H4(螺纹)
    - 注意：禁止修改页签颜色
    - 【部分更新】如果只提取到盘螺或螺纹其中一个价格，另一个为空：
      → 有值的价格必须写入，缺失的保留原值不动
      → 禁止将None/空写入覆盖旧价格
      → 只有盘螺和螺纹都为空时才跳过整个厂家
3.7 【本次新增】库存颜色标注
    ├─ 3.7.0 先保存H列回写结果（wb.save + wb.close），再执行库存颜色
    │        避免wb.save()覆盖库存颜色（保存时序：先H列→再库存颜色→不再重复保存）
    ├─ 3.7.1 清空"报价表"sheet原有颜色
    ├─ 3.7.2 建立映射：
    │         厂家名→Excel列（读取第1行）
    │         规格+长度→行号（读取A-C列）
    ├─ 3.7.3 逐厂家标注单元格：
    │         充足=蓝色 / 告警=黄色 / 缺货=红色
    │         只标第一列（价格列），不标网差列
    └─ 3.7.4 长江特殊：
              "厂内"开头的库存 → 标注T列
              "蚌埠"开头的库存 → 标注V列
```

### 步骤4：OCR提取结果校验与补充文件检查
```
4.1 检查所有 ocr价格提取_*.json 文件的 record_count
    - 对于 record_count=0 的 .txt 文件：
      → 读取原始 .txt 内容，检查是否包含目标城市价格
      → 如果包含但提取失败，报告为"提取异常"
4.2 检查补充.txt文件是否被正确应用
    - 调整值（如"蚌埠下30"）：确认已在基础价格上加减
    - 直接价格（如"蚌埠：3195螺纹"）：确认已覆盖图片价格
    - 如果未被应用，在报告中标注并说明
```

### 步骤5：数据验证
```
5.1 网价链路验证
    - 读取当前项目Excel，检查G1/G3/G4实际值
    - 读取回写前备份，获取老价格值
    - 对比确认价格确实已写入
5.2 图片/文档链路验证
    - 读取当前项目Excel，检查H1/H3/H4实际值
    - 检查"报价表"sheet的库存颜色是否已填充
    - 检查页签颜色是否正确（网价未匹配厂家标红）
5.3 所有值必须从文件读取，禁止推测编造
    - 如果报告JSON未生成，标注"该报告未生成"
    - 如果备份不存在，标注"备份不存在，无法确认老值"
```

### 步骤6：生成报告
```
6.1 生成总结报告 + 分阶段报告
6.2 输出结果概述
    ├─ 网价更新明细（G1/G3/G4）
    ├─ 图片/文档更新明细（H1/H3/H4）
    ├─ 部分更新项（如有）
    ├─ 【新增】库存颜色标注明细（厂家 | 钢材型号和规格 | 库存情况 | 单元格）
    ├─ 汇总统计表
    └─ 报告文件路径
```

## 库存颜色标注详细说明

### 颜色值
| 状态 | 颜色 | 色值 |
|------|------|------|
| 充足 | 蓝色 | `FF0070C0` |
| 告警 | 黄色 | `FFFFC000` |
| 缺货 | 红色 | `FFFF0000` |

### 状态判断标准
| 原始描述 | 判断结果 |
|---------|---------|
| "无货"、"无" | 缺货（红色） |
| "极少"、"少" | 告警（黄色） |
| "XX件"（如82件、166件） | 告警（黄色）- **不是充足** |
| "XX配"（如22E配、25E配） | 告警（黄色） |
| 规格后无任何说明 | 充足（蓝色） |

### 长江特殊映射
| 库存前缀 | 标注列 | 说明 |
|---------|--------|------|
| "厂内" | T列 | 厂内库存 |
| "蚌埠" | V列 | 蚌埠厂库库存 |

---

## 关键阻断点

### 厂家对照待确认
- 发现新厂家时必须阻断，等待用户在对话中确认。
- 确认后才能继续执行回写。
- 已有对照自动复用，不重复确认。

### 数量准确性
- 所有汇总数量必须与明细列表逐项核对一致。
- 不得估算或假设数量。
