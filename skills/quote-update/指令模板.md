# 报价更新指令模板

> 使用方法：复制以下模板，替换【】中的内容后发送给 Agent/大模型。

## 更简单的方式：用脚本生成可复制指令

不想手工改模板时，在项目根目录运行：

```powershell
# 单文件 dry-run 指令
python skills/quote-update/scripts/make_agent_prompt.py single-dry-run `
  --project "项目报价/<实际文件名>.xlsx" `
  --mode both

# 单文件 confirm-write 指令
python skills/quote-update/scripts/make_agent_prompt.py single-confirm `
  --project "项目报价/<实际文件名>.xlsx" `
  --mode both `
  --manifest "运行产物/<项目名>/dry_run_manifest.json"

# 批量 dry-run 指令
python skills/quote-update/scripts/make_agent_prompt.py batch-dry-run `
  --project-dir "项目报价" `
  --glob "*.xlsx" `
  --mode both
```

脚本只生成“发给其他 agent 的提示词”，不会执行报价更新，也不会修改 Excel。

## 模板一：单文件更新（最常用）

```
请更新项目报价文件。

【必读】执行前必须按顺序阅读以下文档，理解完整流程和规则后再开始操作：
1. skills/quote-update/SKILL.md（总流程概览）
2. skills/quote-update/references/workflow.md（完整工作流程）
3. skills/quote-update/references/rules.md（所有关键规则，必须逐条遵守）
4. skills/quote-update/references/commands.md（命令模板和输出格式）

【更新目标】
- 项目文件：【项目报价/安徽合肥-蚌投(春和苑）询价表2026.4.24.xlsx】
- 更新模式：【both】（web=仅网价 / image_doc=仅线下 / both=全部）

【执行步骤，必须严格按序执行，禁止跳过任何步骤】

步骤0：前置准备
  - 从文件名提取地点信息（如"合肥"、"蚌埠"）
  - 确认运行产物目录存在：运行产物/<项目名>/
  - 【重要】文件路径处理：
    先运行 ls "项目报价/" 列出实际文件名，再复制粘贴到命令中。
    文件名中可能含有全角符号（如（）、，），禁止自行转换全角/半角，
    必须使用磁盘上的原始文件名，路径与实际文件名完全一致才能执行成功。

步骤1：网价链路（如果模式是web或both）
  1.1 运行命令：python skills/quote-update/scripts/run_single.py --project "项目报价/【安徽合肥-蚌投(春和苑）询价表2026.4.24.xlsx】" --mode web --headless
  1.2 如果命令输出中出现"待确认"或"pending"：
      → 立即停止，在对话中展示完整待确认明细，等待我确认
      → 我确认后，你写入已确认文件
      → 运行 python skills/quote-update/scripts/check_pending_mapping.py --mapping-json "运行产物/<项目名>/厂家对照表_安徽合肥_待确认.json" 确认待确认为0
      → 然后重新执行1.1的命令
  1.3 网价链路完成后，记录结果（G1/G3/G4的价格变动）

步骤2：图片/文档链路（如果模式是image_doc或both）
  2.1 运行命令：python skills/quote-update/scripts/run_single.py --project "项目报价/【安徽合肥-蚌投(春和苑）询价表2026.4.24.xlsx】" --mode image_doc --headless
  2.2 如果命令输出中出现"待确认"或"pending"：
      → 立即停止，在对话中展示完整待确认明细，等待我确认
      → 处理方式同1.2
  2.3 图片/文档链路完成后，记录结果（H1/H3/H4的价格变动+库存颜色标注明细）

步骤3：结果回报
  - 按README.md第5.0节的格式，完整输出本次更新结果
  - 必须包含：已更新明细表、未更新项及原因、汇总统计表、报告文件路径
  - 价格格式必须是"老值 → 新值"，例如：3690 → 3690
  - 未更新项必须说明具体原因，禁止只写"未更新"

【关键规则提醒，违反任何一条即为执行错误】
1. 图片价格提取：以报价表格区域的价格为准，图片上/下方的差值描述（如"合肥+220"）忽略不计算
2. 报价区域价格为0或空时：输出null，在报告中说明差值描述，禁止自行计算
3. 差值描述 vs 补充文件：图片差值描述不应用；补充.txt的调整规则必须应用
4. 待确认阻断：发现新厂家必须停止，在对话中展示明细等我确认，禁止私自确认
5. 金虹规格：提取螺纹14，其他厂家提取螺纹18
6. 页签颜色：网价先清空后标红未匹配；图片/文档链路禁止改页签颜色
7. 库存颜色：只标第一列（价格列），不标网差列
```

## 低能力模型安全执行规则

1. 禁止手动打开或直接编辑项目报价 Excel。
2. 首次执行必须使用 `--dry-run`，只生成报告和待确认表。
3. 出现 `pending_confirmation`、厂家未确认、价格偏差过大、无网价参考、电议、脚本非零退出码时，必须停止并向用户报告，不得自行绕过。
   - 如果失败原因是网价登录失败、验证码/滑块、登录入口不可见，不得反复重跑 headless；改用不带 `--headless` 的 dry-run，让用户在有头浏览器中完成登录。
   - 有头模式默认等待人工登录 180 秒；需要更长时间时可追加 `--manual-login-timeout 300`。
4. 只有用户明确确认后，才允许用同一命令追加 `--confirm-write` 进行真实写入。
   - `--confirm-write` 默认复用 dry-run 产物直接 apply，不重复抓网价/跑OCR。
   - 仅在需要重生产物时，才追加 `--refresh-web-artifacts` 或 `--refresh-image-artifacts`。
5. 写入完成后必须读取 JSON/Markdown 报告和 audit 结果，不得凭模型记忆汇报。
6. 不得编造价格、库存、厂家映射、更新数量。
7. 禁止创建临时 Python 脚本修改厂家对照表、Excel、库存颜色或报告；只能使用 `skills/quote-update/scripts/` 下的正式脚本。
8. 禁止直接调用 `ocr_price.*` 内部函数补写 Excel 或库存颜色。正式脚本失败时必须停止并报告。
9. dry-run 后必须检查输出的 `Manifest:` 路径；confirm-write 必须带 `--manifest` 使用同一次 dry-run 产物。
10. 最终库存颜色报告必须逐条列出：厂家、钢材型号和规格、库存情况、单元格；禁止只写蓝/黄/红总数。

## 使用说明

### 单文件更新时替换3处：

| 替换项 | 说明 | 示例 |
|--------|------|------|
| 项目文件路径 | 项目报价目录下的xlsx文件名 | `安徽合肥-蚌投(春和苑）询价表2026.4.24.xlsx` |
| 更新模式 | 根据需要选择 | `both` / `web` / `image_doc` |
| 地点信息 | 如果文件名中地点不明显，需额外说明 | 合肥、蚌埠 |

### 批量更新时替换3处：

| 替换项 | 说明 | 示例 |
|--------|------|------|
| 项目文件夹 | 包含xlsx文件的目录 | `项目报价` |
| 文件匹配规则 | glob模式匹配文件名 | `*.xlsx` |
| 更新模式 | 根据需要选择 | `both` / `web` / `image_doc` |

## 常见场景指令速查

| 场景 | 模式 | 单文件命令 | 批量命令 |
|------|------|-----------|---------|
| 全部更新（网价+线下） | both | `--mode both` | `--mode both` |
| 只更新网价 | web | `--mode web` | `--mode web` |
| 只更新线下报价 | image_doc | `--mode image_doc` | `--mode image_doc` |
