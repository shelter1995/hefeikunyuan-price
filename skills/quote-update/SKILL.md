# 报价更新 Skill

## 触发条件
当用户需要更新项目报价（钢材价格）时调用此 Skill。

## 执行流程

### 步骤1：前置准备
1. **文件路径确认**：用 `Glob` 或 `LS` 确认 `项目报价/` 目录下的**实际文件名**，以磁盘上的名字为准，禁止增删字符（`【】` 标记不是文件名的一部分）。
2. 解析用户输入，识别更新模式：
   - 默认 `mode=both`（网价 + 图片/文档）
   - 明确指定"仅网价" → `mode=web`
   - 明确指定"仅图片/文档" → `mode=image_doc`
3. 从文件名提取地点信息：
   - `地点1`：用于网价查询
   - `地点2`：用于图片/文档匹配
4. 创建运行产物目录：`运行产物/<项目报价表名>/`

### 步骤2：网价链路（mode=web 或 both 时执行）

```
1. 登录网价站点
2. 查询最新价格清单（按日期+时间排序）
3. 生成清单Excel
4. 加载/复用厂家对照表
5. 【阻断点】存在新厂家待确认？
   - 是 → 展示明细 → 用户确认 → 写入已确认
   - 否 → 继续
6. 回写项目文件：G1(日期) / G3(盘螺) / G4(螺纹)
7. 页签处理：先清空后标红未匹配
```

### 步骤3：图片/文档链路（mode=image_doc 或 both 时执行）

```
1. 收集线下报价文件（线下报价/*.jpg *.png *.txt）
2. 提取价格和库存（统一使用MiniMax视觉识别）
   ├─ .jpg/.png → MiniMax视觉识别（提取价格+库存）
   │             输出：价格 + 库存状态（充足/告警/缺货）
   ├─ .txt      → 文本解析（提取价格+库存）
   └─ 补充.txt  → 解析调整规则（下30/电议等）
3. 生成JSON结果文件（运行产物/ocr价格提取_*.json）
4. 加载/复用图片文档对照表（已有自动复用）
5. 【阻断点】存在新厂家待确认？
   - 是 → 展示明细 → 用户确认 → 写入已确认
   - 否 → 继续
6. 回写项目文件：H1(日期) / H3(盘螺) / H4(螺纹)
   - 注意：禁止修改页签颜色
7. 库存颜色标注
   ├─ 清空"报价表"sheet原有颜色
   ├─ 建立映射：厂家名→列 / 规格+长度→行
   ├─ 逐厂家标注：充足=蓝 / 告警=黄 / 缺货=红
   └─ 长江特殊：厂内→T列 / 蚌埠→V列
```

### 步骤4：数据验证与报告生成
```
1. OCR提取结果校验：检查 record_count=0 的 .txt 文件是否实际有价格数据
2. 补充文件检查：确认调整规则或直接价格是否已正确应用
3. 数据验证：读取备份+当前Excel，确认老值/新值真实有效
4. 生成总结报告 + 分阶段报告
5. 输出结果概述
   ├─ 网价更新明细
   ├─ 图片/文档更新明细
   ├─ 部分更新项（如有）
   ├─ 库存颜色统计（蓝/黄/红数量）
   ├─ 汇总统计表
   └─ 报告文件路径
```

## 关键规则

### 图片/文档提取（统一使用MiniMax视觉识别）
- **图片文件**：使用 MiniMax 视觉识别提取价格和库存
  - 所有图片（.jpg/.png/.pdf）统一使用MiniMax视觉识别
  - 不再使用PaddleOCR或其他OCR方案
- **文本文件**：正则解析价格+库存
- **补充文件**：解析调整规则（下30/电议等）

### 部分价格更新规则（重要）
- 如果某厂家只提取到盘螺或螺纹其中一个价格，另一个为空/缺失：
  - **必须把有值的价格写入**对应单元格
  - 缺失的价格**不写入**（保留单元格原值），禁止将None/空写入覆盖旧价格
  - 在结果报告中标注"部分更新"并说明缺失项
  - 只有盘螺和螺纹**都为空**时才跳过整个厂家

### 规格提取规则（重要）
- **金虹厂家**：默认提取 **螺纹14** 价格
- **其他厂家**：默认提取 **螺纹18** 价格
- 如果图片中无对应规格 → 设为null（不强制提取）

### 图片价格提取原则
- **以报价表格区域的价格为准**，不自行根据图片上/下方的差值描述计算价格。
- 图片中的差值描述（如"合肥+220"）仅作参考，直接取表格中对应城市的值。
- 报价区域某城市价格为0或空，且上方有差值描述时，价格置空，在报告中说明。
- **差值描述 vs 补充文件**：
  - 图片中的差值描述（如"合肥+220"）→ **不应用**，取表格值
  - 补充.txt文件的调整规则（如"蚌埠下30"）→ **必须应用**

### 库存颜色标注规则
- **执行时机**：图片/文档价格回写完成后
- **目标Sheet**：仅"报价表"sheet
- **操作步骤**：
  1. 清空原有颜色
  2. 建立映射（厂家→列、规格→行）
  3. 标注颜色：充足=蓝 / 告警=黄 / 缺货=红
  4. 只标第一列（价格列），不标网差列

### 库存状态判断
| 原始描述 | 判断结果 |
|---------|---------|
| "无货"、"无" | 缺货（红色） |
| "极少"、"少" | 告警（黄色） |
| "XX件"（如82件） | 告警（黄色） |
| "XX配"（如22E配） | 告警（黄色） |
| 规格后无说明 | 充足（蓝色） |

### 长江特殊处理
| 库存前缀 | 标注列 |
|---------|--------|
| "厂内" | T列（厂内库存） |
| "蚌埠" | V列（蚌埠厂库库存） |

### 厂家对照规则
- 优先复用已确认对照
- 新厂家必须待确认，禁止私自写价
- 已有对照自动复用，不重复确认
- **待确认阻断**：发现新厂家必须立即停止，在对话中展示完整明细后等待用户确认，禁止私自批量确认

### 文件路径确认规则
- 项目文件名必须以磁盘实际名称为准，执行前用 `Glob`/`LS` 确认
- 指令中的 `【】` 标记不是文件名的一部分，禁止照搬进命令参数

### 命令执行失败处理规则
- 任何命令以非零退出码结束时：报告完整错误 → 分析原因 → 提出方案 → **等待用户确认**
- 【禁止】绕过主流程脚本直接调用子命令
- 【禁止】命令失败后编造数据或伪造报告
- 修复后必须重新运行同一个命令

### 数据验证规则
- 所有价格数据必须从实际文件读取，禁止推测编造
- 如果报告JSON未生成，标注"该报告未生成"
- 如果备份不存在，标注"备份不存在，无法确认老值"

### 补充文件处理规则
- 补充.txt文件中的调整规则或直接价格**必须应用**
- 调整值（如"下30"）：在基础价格上加减
- 直接价格（如"蚌埠：3195螺纹"）：覆盖图片中的对应价格
- 如果代码未能正确处理，在报告中标注说明

### OCR提取结果校验规则
- 检查 `ocr价格提取_*.json` 的 record_count
- 对于 record_count=0 的 .txt 文件：读取原始文件，确认是否包含目标城市价格
- 如果包含但提取失败，报告为"提取异常"并建议人工补充

### 价格对比格式
- 必须使用"老值 → 新值"格式
- 例：`3690 → 3690`

## 执行命令

### 单文件
```powershell
python skills/quote-update/scripts/run_single.py `
  --project "项目报价/<项目Excel路径>" `
  --mode both `
  --headless
```

### 批量
```powershell
python skills/quote-update/scripts/run_batch.py `
  --project-dir "项目报价" `
  --glob "*.xlsx" `
  --mode both `
  --headless
```
